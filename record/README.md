# The record layer

The sanitized distillate of every X2 project extraction (the x2:extract loop). Only method-level
lessons live here: patterns, rules, figures, and rulings, in domain-neutral language. The raw
evidence (ledgers, decision files, names) stays in each project's own private repository and never
enters this tree. Nothing lands here without a human ruling; rejected items are recorded as ruled
so they do not resurface.

Projects are identified as P1, P2, ... in extraction order. P1 is the pilot from the kernel
acceptance test.

- `negative-space.md`: the negative-space register and the cross-project churn table. Decompose
  reads the table as a pre-flight: the decisions history says will churn.
- `candidates.md`: accepted candidates queued for the next invariants pass. Items graduate into
  `kernel/claims/` with mechanisms and leave this file.
- `confirmations.md`: claims and disciplines that held under real load, closed to reopening
  without new evidence.
- `metrics.md`: the metric history under the one counting rule.
